# نظام إدارة محل المستهلكات الطبية ومحاليل المعامل

نظام ويب متكامل (Web App) يعمل من أي جهاز عبر رابط واحد. يشمل:

- تسجيل دخول لعدة مستخدمين، وكل مستخدم له صلاحيات يحددها المدير (عرض/تعديل فقط لأقسام معينة).
- **الأصناف**: كود، اسم، تصنيف، وحدة، سعر شراء، سعر بيع، كمية، حد أدنى للتنبيه.
- **المشتريات**: تسجيل فواتير شراء من الموردين، تزيد المخزون تلقائياً.
- **المبيعات / كاشير**: نقطة بيع سريعة، تخصم من المخزون تلقائياً، وتمنع البيع بأكثر من المتوفر.
- **المخزون**: عرض الأرصدة الحالية + سجل كل حركة (شراء/بيع/تعديل جرد) + تنبيه بالأصناف التي وصلت للحد الأدنى.
- **المستخدمين والصلاحيات**: المدير يضيف مستخدمين ويحدد صلاحيات كل واحد بشكل مرن (checkbox لكل خاصية).
- **سجل التعديلات (Audit Log)**: يسجل من أضاف/عدّل/حذف أي شيء ومتى، لكل العمليات في النظام.

النظام مبني بـ Next.js (يعمل على الموبايل والكمبيوتر من نفس الرابط) + قاعدة بيانات PostgreSQL، وقابل للتوسعة مستقبلاً (يمكن إضافة فروع متعددة، تقارير، طباعة فواتير، إلخ) لأن الكود منظم وواضح.

---

## لماذا يجب عليك أنت تنفيذ خطوات النشر؟

لا أستطيع إنشاء حسابات نيابة عنك (تحتاج بريدك الإلكتروني وموافقتك)، ولا أملك اتصال إنترنت من بيئتي لأرفع الكود مباشرة. لكني جهزت كل شيء بحيث تستغرق خطوات النشر منك **10-15 دقيقة فقط**، بدون أي خبرة برمجية، فقط نسخ ولصق.

## خطوات النشر (مرة واحدة فقط)

### 1) أنشئ قاعدة بيانات مجانية (Neon)
1. اذهب إلى https://neon.tech وسجل حساب مجاني (يمكن بحساب Google).
2. أنشئ مشروع جديد (Create Project).
3. من صفحة المشروع، انسخ **Connection String** (يبدأ بـ `postgresql://...`).

### 2) ارفع الكود إلى GitHub
1. أنشئ حساب على https://github.com إن لم يكن لديك.
2. أنشئ مستودع (Repository) جديد فارغ، واسمه مثلاً `medistore`.
3. ارفع مجلد المشروع الذي أرسلته لك إلى هذا المستودع (يمكنك سحب وإفلات الملفات من واجهة GitHub في المتصفح مباشرة عبر "Add file → Upload files" إن لم تكن تستخدم Git من قبل).

### 3) انشر المشروع على Vercel
1. اذهب إلى https://vercel.com وسجل دخول بحساب GitHub.
2. اضغط **Add New → Project** واختر المستودع `medistore`.
3. في صفحة الإعدادات قبل النشر، أضف متغيرات البيئة التالية (Environment Variables):
   - `DATABASE_URL` = الرابط الذي نسخته من Neon
   - `JWT_SECRET` = أي نص عشوائي طويل (مثلاً: `a8f9d7s6f9d8g7h6j5k4l3`)
   - `SETUP_SECRET` = كلمة سر مؤقتة من اختيارك (مثلاً: `firstsetup123`)
4. اضغط **Deploy** وانتظر دقيقتين. سيقوم Vercel تلقائياً بإنشاء جداول قاعدة البيانات أثناء البناء.
5. بعد انتهاء النشر ستحصل على رابط مثل: `https://medistore-xxxx.vercel.app` — هذا هو رابط النظام الذي تفتحه من أي جهاز (موبايل أو كمبيوتر).

### 4) أنشئ حساب المدير الأول
افتح هذا الرابط مرة واحدة فقط (يمكنك استخدام أي أداة إرسال طلبات، أو حتى من متصفح الموبايل عبر موقع مثل https://reqbin.com):

```
POST https://your-app.vercel.app/api/setup
Content-Type: application/json

{
  "secret": "firstsetup123",
  "name": "اسمك",
  "username": "admin",
  "password": "كلمة-مرور-قوية"
}
```

بعدها افتح رابط النظام وسجل دخول باسم المستخدم وكلمة المرور اللي حددتها. من داخل النظام (صفحة "المستخدمين والصلاحيات") يمكنك إضافة بقية الموظفين وتحديد صلاحيات كل واحد.

> ملاحظة أمان: بعد إنشاء حساب المدير الأول، يفضّل تغيير قيمة `SETUP_SECRET` في إعدادات Vercel حتى لا يستخدمها أحد غيرك لاحقاً.

---

## هل يمكن تطويره لاحقاً؟

نعم، الكود منظم في مجلدات واضحة (`app/api` للخادم، `app/(dashboard)` للواجهات) بحيث يسهل إضافة:
- فروع متعددة للمحل
- طباعة فواتير PDF
- تقارير أرباح مفصّلة
- ربط باركود
- إشعارات تلقائية عند نفاد صنف

فقط أرسل لي لاحقاً أي إضافة تريدها وسأطورها على نفس الأساس.

## التشغيل محلياً (اختياري - لمن يريد تجربة الكود على جهازه)

```bash
npm install
cp .env.example .env   # ثم عدّل القيم داخل .env
npx prisma db push
npm run dev
```

ثم افتح `http://localhost:3000`.
